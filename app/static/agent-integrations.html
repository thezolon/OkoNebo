AI Integration Guide

Use OkoNebo as a weather source for AI agents using MCP, ACP, or direct REST.

Two Ways To Integrate

Auto-config profile (for agent runtimes): /.well-known/okonebo-agent.json
Manual instructions (for humans): /agent-manual.html
Plain-text prompt block (copy/paste): /.well-known/okonebo-agent-instructions.txt

Use auto-config when possible. Use manual mode when you want to inspect or customize every instruction.

Step 1: Create an Agent Token

  1. Open Admin → AI Agent Tokens.
  2. Create a token with only the scopes your agent needs.
  3. Copy the token immediately (shown once).

Recommended default scopes: weather.read, config.read, stats.read.

MCP (Model Context Protocol)

The repository includes an MCP adapter at scripts/mcp_server.py.

export OKONEBO_BASE_URL=http://localhost:8888
export OKONEBO_AGENT_TOKEN=<your_agent_token>
python scripts/mcp_server.py

Exposed tools: get_current, get_forecast, get_hourly, get_alerts, get_metar, get_tides, get_stats, get_config, get_bootstrap, get_capabilities.

ACP (Agent Communication Protocol / OpenClaw)

Use the same token model and endpoint map as MCP. Start with read-only weather tools.

Base URL: http://localhost:8888
Auth: Authorization: Bearer <agent_token>
Capabilities endpoint: GET /api/capabilities

If your ACP runtime supports custom tool definitions, map each tool to the REST endpoints listed below.

Direct REST (Any Agent Framework)

All major frameworks can call these endpoints directly with bearer auth.

GET /api/current - current observations
GET /api/forecast - multi-day forecast
GET /api/hourly - hourly forecast
GET /api/alerts - active weather alerts
GET /api/metar - aviation METAR
GET /api/tides?days=2 - tide predictions
GET /api/stats - provider call counters
GET /api/config - location/config metadata
GET /api/bootstrap - first-run/auth state
GET /api/capabilities - tool + scope map

Security Recommendations

  • Issue one token per agent system (do not share tokens).
  • Use minimum scopes only.
  • Rotate tokens on schedule or when users/services change.
  • Revoke immediately if compromised.
  • Keep OkoNebo private behind your network/VPN when possible.
