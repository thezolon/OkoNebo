This page explains what administrators can configure and what each feature controls.

Runtime Setup

  • Home/work location controls the map center, monitored alert regions, and default forecast point.
  • Timezone affects local display formatting in dashboard and timeline elements.
  • User Agent is sent to upstream providers that require application identification.
  • Map Provider selects the default base map style for the main dashboard.

Provider Controls

  • Enable/disable providers at runtime without rebuilding the app.
  • Pull cycle (seconds) controls how often we check each provider for updates. Lower = more responsive but more API calls. Default is 5m for most providers.
  • API keys saved in setup are stored encrypted in secure settings storage.
  • Test each provider individually to verify credentials work, or use Test All to quickly validate all enabled providers.
  • Keyless providers (NWS, AviationWeather, NOAA Tides) can remain enabled by default for baseline coverage.
  • If one provider fails, weather routes fall back to others in configured order.

Auth and Access

  • Enable Auth to require user login for protected operations.
  • Viewer Login requirement forces read-only users to authenticate for normal viewing.
  • Admin user credentials control write operations such as settings and token management.
  • In no-auth mode, local admin functions are intentionally open for single-user deployments.

AI Agent Tokens

  • Create one token per agent system with least-privilege scopes.
  • Token value is shown once on creation, then only metadata is retained in the UI.
  • Revoke tokens immediately when services/users rotate or compromise is suspected.
  • Use the integration guide for MCP/ACP/REST connection models.

Admin Panel Tips

  • Unsaved changes indicator (yellow circle ⊙) shows when you've modified settings but haven't saved yet.
  • Save Settings button persists all changes to the encrypted database. Press Ctrl+S (or Cmd+S on Mac) as a keyboard shortcut.
  • Discard Changes reverts all unsaved edits back to the last saved state—useful if you change your mind.
  • Copy Token button appears after creating an agent token. Click once to copy to clipboard.
  • Delete confirmation prevents accidental token removal—you'll be asked to confirm before deletion.
  • Settings are organized into clearly labeled cards for easy scanning.

Operational Tips

  • Back up runtime databases before large config changes.
  • After changing environment variables, rebuild/restart containers to apply them.
  • Use System Status + Ops Timeline on the dashboard to validate behavior after changes.
